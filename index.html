PDO - A Conectividade em larga escala


Para que serve?

PDO (PHP DATA OBJECTS) é um módulo desenvolvido para realizar a comunicação com um banco de dados relacional. Este módulo segue o paradigma orientado a objeto, tendo seu surgimento a partir da versão 5 do php.



Utilidades

O PDO possui diversas utilidades, entre as quais podemos destacar:

-> Maior comunicação com diversos SGBDS.
-> Segurança no envio de informações.
-> Sintaxe Simples e fácil de aprender.
-> Acesso integrado ao banco.
-> Disponiblidade de recursos para diversas operações.


Conexão

A conexão deve ser realizada através de algumas funções já definidas da classe pdo.


Passo 1: Deve ser criada uma função para conectar.
Passo 2: Colocar os blocos try - catch para caso ocorra a existência de algum erro.
Passo 3: Dentro do bloco try colocamos uma variável que armazenará a nossa conexão, neste caso a variável $pdo;
Ela receberá uma instância da classe PDO com alguns parâmetros:

-> sgbd:host=servidor;dbname=nome_do_banco, usuario, senha
Passo 4: No bloco catch colocamos a PDOException com uma variável que receberá a exceção, depois colocamos a função getMessage() para alertar o erro.
Passo 5: Coloca-se um return na variável $pdo para retornar a conexão.

Métodos

bindValue

Esta função serve para receber um valor para ser colocado em uma instrução SQL podendo receber uma variável de referência ou então um valor já predefinido.

bindParam

No caso da função bindParam é a mesma coisa, porém esta função não aceita valores já definidos, só aceita variáveis de referência.

Prepare

A função prepare serve para preparar uma string SQL, ou seja nela colocamos o comando sql que desejamos realizar em nosso banco.

Execute

Após a utilização do prepare deve-se utilizar uma função para executar a operação que queremos realizar no banco, esta função é o execute.
